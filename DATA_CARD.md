# Data Card

이 저장소는 목적이 다른 두 데이터 자산을 분리해서 다룹니다. 둘을 하나의 모델 학습 데이터처럼 표현하지 않는 것이 이 포트폴리오의 핵심 원칙입니다.

## 1. 앱 데모용 합성 수요 데이터

| 항목 | 내용 |
| --- | --- |
| 파일 | `data/raw/retail_sales.csv` |
| 생성 코드 | `backend/data_generator.py` |
| 크기 | 36,500개 일별 관측치 (5개 매장 × 10개 상품 × 730일) |
| 기간 | 2023-01-01부터 730일 |
| 주요 변수 | 날짜, 매장 ID, 상품 ID, 판매량, 프로모션 여부, 요일 |
| 재현성 | 난수 시드 42를 고정해 동일한 데이터를 생성 |
| 목적 | Docker 환경에서 즉시 실행 가능한 예측 UI·API·MLflow 데모 |

생성 과정은 매장/상품별 기본 수요, 주간·연간 계절성, 완만한 추세, 임의 프로모션 효과와 노이즈를 결합합니다. 이 데이터는 실제 고객·거래·상품 정보를 포함하지 않습니다.

### 사용 제한

- 실제 수요나 매출을 대표하지 않으므로 재고·발주 의사결정에 사용하면 안 됩니다.
- 프로모션은 확률적으로 생성됐으며 실제 캠페인 정책을 반영하지 않습니다.
- `is_promo` 외의 가격·재고·휴일·품절·채널 같은 실무 핵심 변수가 없습니다.

## 2. Commax 출하 데이터 EDA 사례

| 항목 | 내용 |
| --- | --- |
| 분석 노트북 | `notebooks/EDA/final_eda.ipynb` |
| 용도 | 실데이터 기반의 데이터 품질·품목 집중도·계절성·외부 지표 탐색 |
| 데이터 범위 | 20,096행 × 240열, 2015-01부터 2025-08까지의 월 단위 출하 분석 |
| 사용 권한 | 포트폴리오 공개 사용 가능 여부를 데이터 제공자에게 확인함 |
| 원본 파일 | 이 저장소에는 재배포하지 않음 |

이 분석 사례는 앱 데모의 Prophet 학습에 사용되지 않습니다. 원본 데이터가 저장소에 없으므로 노트북을 완전 재실행하려면 권한 있는 원본 파일 `Final_KR_modeling_long_with_external_data.csv`를 별도로 제공해야 합니다.

EDA에서 확인한 수치는 [COMMAX_EDA_CASE_STUDY.md](COMMAX_EDA_CASE_STUDY.md)에 해석·주의점과 함께 정리했습니다.
